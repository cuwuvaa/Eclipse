# Eclipse - Project Checklist

Это дорожная карта для разработки игрового мессенджера "Eclipse", аналога Discord (еще круче).

## Основные функции (MVP)
- [$] Аутентификация пользователя (регистрация, вход, выход)
- [$] Профили пользователей (имя пользователя, отображаемое имя, аватар)
- [$] Создание/присоединение к "Комнатам"
- [$] Текстовые каналы внутри комнат
- [$] Обмен текстовыми сообщениями в реальном времени в каналах
- [$] Голосовые каналы внутри комнат
- [$] Базовый голосовой чат в реальном времени в каналах
- [$] Список комнат/серверов, к которым принадлежит пользователь
- [$] Список пользователей в комнате

## Бэкенд (Django)
- [$] Настройка проекта Django и приложений (`EclipseUser`, `EclipseRoom` уже существуют)
- [$] Модель пользователя (`EclipseUser`, похоже, готова)
- [$] Модель Комнаты/Сервера
- [$] Модель Сообщения (с внешними ключами к участнику)
- [$] REST API для всех основных действий (отправка сообщений, создание комнат и т.д.) с использованием Django Rest Framework
- [$] Обработка WebSocket для сообщений в реальном времени (с использованием Django Channels)
- [$] Сигнальный сервер WebRTC для голосового чата (может быть частью настройки Django Channels)

## Фронтенд
- [ ] Базовый макет пользовательского интерфейса (боковая панель для серверов, список каналов, главное окно чата)
- [$] Страницы входа/регистрации
- [$] Страница/модальное окно профиля пользователя
- [$] Интерфейс чата для текстовых каналов (отображение сообщений, поле ввода)
- [$] Пользовательский интерфейс для присоединения/выхода из голосовых каналов
- [$] Отображение пользователей в голосовых/текстовых каналах
- [$] Использование современного фронтенд-фреймворка (например, React или Vue.js) или Django templates с JavaScript
- [$] Подключение к WebSocket для обновлений в реальном времени
- [$] Логика WebRTC на стороне клиента для обработки голосовых потоков

## Реализация голосового и текстового чата
- [%] **Текстовый чат**:
    - [$] Реализовать потребителей WebSocket в Django Channels
    - [$] JavaScript на фронтенде для подключения к WebSocket
    - [$] Трансляция полученных сообщений всем пользователям в канале
    - [$] Хранение сообщений в базе данных
- [%] **Голосовой чат**:
    - [$] Реализовать механизм сигнализации с использованием WebSockets (для согласования сеанса WebRTC)
    - [$] Обработка кандидатов SDP (Session Description Protocol) и ICE (Interactive Connectivity Establishment)
    - [$] JavaScript на фронтенде для получения доступа к микрофону
    - [$] Создание и управление объектами `RTCPeerConnection`
    - [$] Функциональность отключения звука/наушников

## Модерация комнат 
- [$] Добавить 3 базовые роли - создатель, модератор, пользователь
- [ ] Создать действия для комнаты
- [ ] Реализовать соответствующие действия для ролей
- [ ] Управление ролями и разрешениями

## Развертывание и инфраструктура

- [ ] Настройка сервера Redis для Django Channels
